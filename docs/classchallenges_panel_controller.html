Detailed Description

Class that controls the panel that shows all the challenges that the current user has, it also controls the image that will be shown when the user doesnt have any challenge.

Member Function Documentation

◆ adjustPanelSize()

void challengesPanelController.adjustPanelSize ( )
private

This method should be called each time that the panel adds or quits an element. This method adjust the height of the panel to keep the apparence when the number of elements changes.

◆ Awake()

void challengesPanelController.Awake ( )
private

This method is called before the first frame, it instanciate the challenges_, panelFilled_ and lastCount_ properties. And if the userDataIsReady method of firebaseHandler class returns true it calls the fillPanel method.

◆ fillPanel()

void challengesPanelController.fillPanel ( )
private

This method is called to instanciate the panel. It creates a prefab for each new challenge invitation that the current user has. It changes the panelFilled_ property to true.

◆ Update()

void challengesPanelController.Update ( )
private

This method is called once per frame, it checks if the panel has al ready filled and if the userDataIsReady method of firebaseHandler class returns true it calls the fillPanel method. If the lastCount_ property isnt equal to the current challenges_ count it calls the adjustPanelSize method. It also shows the noChallengesImage_ gameObject if the challenges_ list is empty.
